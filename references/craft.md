# Writing Craft — 写作内功

去AI味是地板。这个文件是天花板。

这里的原则不分中英文、不分平台、不分时代。鲁迅、李白、Paul Graham、Naval、半佛仙人——风格天差地别，但底层规律相同。

---

## 原则 1: 密度 > 长度

一句废话都是对读者时间的犯罪。

### 核心理念

好文字是压缩的。不是"用更少的字说同样的话"，而是"用同样的字塞进更多的信息和情感"。

- 鲁迅："从来如此，便对么？" — 7个字，挑战了整个社会的惯性思维
- Naval: "Specific knowledge is found by pursuing your genuine curiosity." — 一句话顶一本职业规划书
- 李白："抽刀断水水更流，举杯消愁愁更愁" — 14个字，两层递进的无奈

### 压缩技术

**删除法：逐句问"删了这句，文章是否变差？"**

- 如果答案是"不变" → 删
- 如果答案是"变好了因为更紧凑" → 删
- 只有"变差了因为缺了关键信息/情感" → 留

**合并法：两句弱句合成一句强句**

| Before (两句弱)                                             | After (一句强)                                                 |
| ----------------------------------------------------------- | -------------------------------------------------------------- |
| 创业很难。很多人都失败了。                                  | 创业的默认结局是失败。                                         |
| I worked really hard. Eventually things started to improve. | I ground through 14-hour days until the numbers finally moved. |
| 这个方法很有效。我试了之后效果很好。                        | 试完当天转化率翻倍。                                           |

**替换法：虚词→实词，抽象→具象**

| 虚          | 实                        |
| ----------- | ------------------------- |
| 很长时间    | 11个月                    |
| 赚了很多钱  | 月入8万                   |
| worked hard | shipped 3 features a week |
| 效果很好    | 退货率从12%降到3%         |

### 密度自测

数一下每段的"信息单元"（一个独立的事实、观点、或情感）。

- < 2个/段 → 太稀，合并或删除这段
- 2-3个/段 → 正常
- 4+个/段 → 检查是否需要拆分（密度高但别让读者窒息）

## 原则 2: 具体→普遍的跳跃

最好的文字是一只脚踩在泥土里，另一只脚踩在云端。

### 核心理念

不是"讲大道理"，也不是"讲小故事"。是**从一个极其具体的细节，一跳跳到一个普遍真理**。这个跳跃就是文章的灵魂。

- 鲁迅写孔乙己在柜台上排出九文大钱 → 跳到整个社会对落魄读书人的冷漠
- Gladwell 写一个推销员的微表情 → 跳到人类说服力的底层机制
- Paul Graham 写 Y Combinator 的一次晚餐对话 → 跳到创业公司为什么不该过早招人

### 跳跃的结构

```
[锚点] 一个极其具体的场景/细节/数字
   ↓
[桥梁] "这件小事让我意识到..." / "后来我才明白..."
   ↓
[起飞] 一个比这个场景大得多的洞察/真理
```

### 实操方法

**Step 1: 先写锚点**

- 必须有感官细节：时间、地点、人物、动作、对话
- 越小越好：不是"创业很难"，而是"凌晨2点我盯着银行余额还剩$847"

**Step 2: 问自己"所以呢？"连问3次**

- 银行余额$847 → 所以呢？→ 我可能发不出下月工资
- 发不出工资 → 所以呢？→ 我第一次理解了什么叫"责任不是选择题"
- 责任不是选择题 → 所以呢？→ 大多数人对"创业风险"的理解停留在PPT上，真正的风险是凌晨2点的那种恐惧

第三个"所以呢"通常就是你要的洞察。

**Step 3: 检验跳跃**

- 锚点够具体吗？（能在脑中成像？）
- 洞察够普遍吗？（不只是你的事，是很多人的事？）
- 跳跃够大吗？（从个人到群体？从事件到规律？从现象到本质？）
- 桥梁够自然吗？（读者是"嗯确实"还是"等一下你怎么跳到这的"？）

### 跳跃的高度层级

| 高度 | 从→到                   | 例子                                        |
| ---- | ----------------------- | ------------------------------------------- |
| 低跳 | 个人经验→个人教训       | "我踩了这个坑，你别踩"                      |
| 中跳 | 个人经验→群体规律       | "我踩了这个坑，其实80%的人都会踩"           |
| 高跳 | 个人经验→人性/社会/时代 | "我踩了这个坑，因为我们这代人被教育成了..." |

一篇好文章至少有一个**中跳**，最好有一个**高跳**。

## 原则 3: 认知落差

读者读完后看世界的角度应该和读之前不同。

### 核心理念

不是给读者新信息（"今天我学到了X"），而是给读者**新框架**（"原来可以这样看待X"）。

- Paul Graham: "Keep your identity small" — 不是新信息，是新的看事物的方式
- 半佛仙人: 把商业骗局拆解成"收智商税的N种变体" — 给了一个从此能识别所有骗局的框架
- Naval: "Play long-term games with long-term people" — 一句话重构了如何选择人生策略

### 认知落差设计

**Step 1: 找到读者的"当前认知"**

- 关于这个话题，大多数人现在怎么想？
- 这个想法哪里有漏洞？哪里过于简化？

**Step 2: 设计"目标认知"**

- 你希望读者读完后怎么想？
- 新认知和旧认知之间的差距有多大？

**Step 3: 设计落差体验**

| 落差类型 | 机制                     | 例子                                      |
| -------- | ------------------------ | ----------------------------------------- |
| 颠覆型   | 旧认知是错的             | "努力不是成功的因素，选择才是"            |
| 升维型   | 旧认知不够完整           | "你以为是能力问题，其实是认知问题"        |
| 连接型   | 两个看似无关的事其实相关 | "健身和投资的底层逻辑是一样的"            |
| 还原型   | 复杂的事其实很简单       | "所有商业模式归根结底只有3种"             |
| 复杂化型 | 看似简单的事其实很复杂   | "你以为会说话就够了？沟通比你想的难100倍" |

**Step 4: 验证落差**

- 读者读第一段和最后一段，认知有变化吗？
- 如果删掉中间所有内容只留首尾，读者会感到"跳跃太大" → 好，说明中间内容在搭桥
- 如果删掉中间读者没感觉 → 认知落差不够大，重新设计

## 原则 4: 节奏感

文字有音乐性。长短、快慢、紧松、重轻——交替出现。

### 中文节奏规则

**短句轰炸 + 长句收住**

```
创业第一年。没钱。没人。没方向。
但就是那种什么都没有的状态，逼出了我后来最值钱的能力——在混沌中做决定。
```

**三短一长法则**：连续2-3个短句（<10字）后，接一个长句（20-40字）。这是中文最自然的呼吸节奏。

**断句制造紧张**

| 正常                       | 有节奏                                   |
| -------------------------- | ---------------------------------------- |
| 那天晚上我做了一个决定     | 那天晚上。我做了一个决定。               |
| 他说了一句话改变了我的想法 | 他说了一句话。就一句。我的想法彻底变了。 |

**句式变化矩阵**

不要连续用同一种句式。交替使用：

- 陈述句："转化率涨了3倍。"
- 反问句："这难道不值得试试？"
- 感叹/独白："我当时真的慌了。"
- 对话引用："他跟我说：'你疯了吧。'"
- 省略句："结果呢？"

**规则**：连续3句同一句式 = 节奏死了。必须切换。

### 英文节奏规则

（见 de-ai-checklist.md English Rhythm 部分）

核心相同：Short punch. Short punch. Then a longer sentence that lets the reader breathe. 中英文的呼吸节奏是通用的。

### 段落节奏

不只是句子内部，段落之间也要有节奏：

```
[重段] 核心论点，有数据有案例，4-5句 — 信息密集
[轻段] 1-2句的感受、吐槽、或转折 — 让读者喘口气
[重段] 下一个论点
[一句段] 就一句话。独立成段。强调。
[重段] 继续
```

**禁止**：连续3个重段（读者窒息），连续3个轻段（内容太水）。

## 原则 5: 金句工程

每篇文章需要至少一个"脱离上下文依然成立"的句子。

### 核心理念

金句是传播的最小单位。读者可能忘掉你的整篇文章，但会记住那一句话、截图那一句话、分享那一句话。

- "不自由，毋宁死。"
- "Stay hungry, stay foolish."
- "种一棵树最好的时间是十年前，其次是现在。"
- "The best time to start was yesterday. The second best time is now."

### 金句的结构模式

**模式 1: 对立统一**
把两个看似矛盾的概念压缩进一句话。

- "慢慢来，比较快。"
- "The things you own end up owning you."
- "最大的冒险是不冒任何险。"

公式：`A的反面其实是A` 或 `A导致了-A`

**模式 2: 重新定义**
用一句话重新定义一个概念。

- "自律就是在没人看的时候，依然做该做的事。"
- "Wealth is assets that earn while you sleep."
- "真正的自信不是'我很好'，是'我不好也没关系'。"

公式：`X不是Y，X是Z`（Z是出人意料的定义）

**模式 3: 类比压缩**
用一个比喻把复杂道理压缩成一个画面。

- "人生不是轨道，是旷野。"
- "Reading is downloading other people's experiences into your brain."
- "焦虑就是在未来还没发生的事情上提前支付情绪利息。"

公式：`抽象概念 = 具象画面`

**模式 4: 悖论/反常识**
说一句读者本能想反驳，但细想又对的话。

- "你不需要更多信息，你需要更少选项。"
- "The best founders are the laziest — they refuse to do anything that doesn't scale."
- "有时候放弃比坚持需要更大的勇气。"

公式：`违反直觉的断言`（但必须有道理）

**模式 5: 数字锚定**
用精确数字让抽象观点可感知。

- "人生就那么3万天，你已经用掉了1万天。"
- "99% of advice is about doing more. The best advice is about doing less."
- "大多数人高估了一年能做的事，低估了十年能做的事。"

公式：`数字 + 对比/反差`

### 金句生产流程

1. **写完初稿后**，标记3-5个你认为是"核心洞察"的句子
2. 对每个候选句子，尝试用上面5种模式重写
3. **独立测试**：把这句话单独拿出来发朋友圈/推特，不加上下文，有人点赞吗？
4. **记忆测试**：合上文章，你还记得这句话的原话吗？如果你自己都记不住，读者更记不住
5. 选出最强的1-2句，确保它们在文章中的位置显眼（段首或段尾，不要埋在中间）

### 金句在文章中的位置

- **开头**：作为hook的一部分（最强位置）
- **转折处**：旧认知→新认知的桥上（第二强位置）
- **结尾**：最后一句话（第三强位置，读者最后记住的）
- **绝不能**：埋在长段落中间（没人看到）

## 原则 6: 广度——让圈外人也想读

深度让专家认可，广度让普通人也能进来。

### 核心理念

一篇文章如果只有目标受众能看懂，它的传播天花板就是那个圈子。**最好的内容是"专业人士觉得有深度，外行人也觉得有意思"。**

- Gladwell 写心理学，不需要心理学学位也能读
- 半佛仙人 写金融骗局，完全不懂金融的人也看得津津有味
- Paul Graham 写创业，上班族也觉得有收获

### 广度的三个层次

**层次 1: 入口无门槛**

- 开头用"人人都有过的经历"入场，不要用专业术语入场
- "你有没有过这种感觉：XX" > "在XX领域，有一个现象值得关注"
- 读者在第一段就应该觉得"这是写给我的"，不管他是什么背景

**层次 2: 跨界类比搭桥**

- 用目标受众圈外的比喻解释圈内的概念
- 投资的道理用做饭来解释，编程的逻辑用装修来类比
- 每用一个专业概念，立刻跟一个日常类比
- 公式：`{专业概念}，说白了就是{生活中的XX}`

**层次 3: 普世情感连接**

- 不管多专业的话题，底层都连接到人类共通的情感：
  - 恐惧（怕失去、怕落后、怕选错）
  - 希望（变好、变强、被认可）
  - 好奇（为什么？怎么做到的？然后呢？）
  - 归属（原来不只是我这样）
  - 愤怒（不公平、被骗、被低估）
- 把专业内容锚定到这些情感上，圈外人就能共鸣

### 广度自测

1. **妈妈测试**：把你的文章给一个完全不懂你领域的人看前3段，他能看下去吗？
2. **跨圈转发测试**：一个不在你目标受众里的人会转发吗？
3. **去掉专业术语测试**：把所有专业词替换成日常用语，文章还成立吗？如果不成立，说明你依赖术语撑内容。

### 广度 vs 深度：不是二选一

```
         广度高
          │
    ┌─────┼─────┐
    │ 大众爆文 │ 传世之作 │
    │（10w+阅读） │（被引用多年）│
────┼─────┼─────┼──── 深度
    │  水文   │ 小众精品 │
    │（谁都能写） │（圈内叫好） │
    └─────┼─────┘
          │
         广度低
```

- 水文 = 既不深也不广（淘汰）
- 大众爆文 = 广度高但深度一般（流量大但不持久）
- 小众精品 = 深度高但广度低（圈内认可但出不了圈）
- **传世之作 = 广度+深度都高**（目标）

大部分内容至少做到"大众爆文"象限。如果话题允许，冲"传世之作"。

## 六个原则的协同

这6个原则不是独立的checklist，它们互相加强：

```
密度 → 逼你压缩每句话 → 自然产生节奏变化
具体→普遍 → 制造认知落差 → 跳跃的瞬间产生金句
节奏 → 控制读者的注意力 → 把金句放在节奏的"重拍"上
认知落差 → 给读者分享动机 → 金句成为传播载体
金句 → 被引用传播 → 带来新读者 → 循环
广度 → 让圈外人也能进入 → 扩大所有其他原则的辐射范围
```

**在 Draft 步骤中的应用顺序**：

1. 先确保有认知落差（方向对不对？）
2. 再做具体→普遍的跳跃（核心素材有没有？）
3. 检查广度（圈外人看得懂前三段吗？）
4. 然后压缩密度（每句话都值得存在吗？）
5. 调整节奏（读起来有没有音乐性？）
6. 最后打磨金句（有没有一句封神？）

**在 Review 步骤中的检验**：

- 密度：删掉任意一段，文章变差了吗？
- 跳跃：有没有从泥土到云端的瞬间？
- 落差：读者读前读后，认知变了吗？
- 广度：不懂这个领域的人能被前三段吸引吗？
- 节奏：朗读出来，有停顿和加速的变化吗？
- 金句：合上文章，还记得哪句话？
