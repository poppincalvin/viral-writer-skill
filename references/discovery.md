# Hot Topic Discovery

## Overview

发现值得写的话题，比写好一篇文章更重要。这个环节解决"写什么"的问题。

核心原则：**不是所有热点都值得追，只追你能赢的热点。**

---

## Step 1: 多源扫描

根据目标语言和平台，选择对应信息源执行 WebSearch。

### 中文信息源

| 信息源           | 搜索模板                                      | 适合领域               |
| ---------------- | --------------------------------------------- | ---------------------- |
| 微博热搜         | `微博热搜 今日 {领域}`                        | 社会、娱乐、情绪型话题 |
| 知乎热榜         | `知乎热榜 {领域} site:zhihu.com`              | 深度、观点、专业话题   |
| 百度热搜         | `百度热搜 今日`                               | 大众话题、全民关注     |
| X/Twitter 中文圈 | `site:x.com {领域关键词} lang:zh`             | 科技、海外华人、留学   |
| 小红书           | `小红书 热门话题 {领域}`                      | 生活方式、消费、情感   |
| 即刻             | `即刻 热门 {领域}`                            | 科技、创业、产品圈     |
| 36氪/虎嗅        | `site:36kr.com OR site:huxiu.com {领域} 今日` | 商业、科技、创业       |

### 英文信息源

| Source        | Search Template                     | Best For                       |
| ------------- | ----------------------------------- | ------------------------------ |
| X/Twitter     | `site:x.com {topic} trending`       | Tech, culture, hot takes       |
| Hacker News   | `site:news.ycombinator.com {topic}` | Tech, startups, engineering    |
| Reddit        | `site:reddit.com {topic} popular`   | Niche communities, debates     |
| Google Trends | `Google Trends {topic} rising`      | Trend velocity, comparisons    |
| LinkedIn      | `site:linkedin.com {topic} viral`   | Professional, career, business |
| Substack      | `site:substack.com {topic} popular` | Long-form, newsletters         |

### 扫描策略

- **广撒网**：至少搜 3 个信息源，每个源取 Top 3-5 话题
- **交叉验证**：同一话题出现在 2+ 个源 = 高热度信号
- **领域过滤**：用用户的领域关键词收窄，不看无关噪音

## Step 2: 趋势阶段判断

对每个候选话题，判断它处于哪个阶段：

| 阶段   | 特征                                   | 写作策略                                             |
| ------ | -------------------------------------- | ---------------------------------------------------- |
| 萌芽期 | 搜索结果少、内容新（<24h）、讨论刚起来 | **最佳时机**。抢先写，定义话题框架，吃第一波流量     |
| 爆发期 | 搜索结果多、各平台都在讨论、大V已入场  | **需要差异化角度**。正面硬刚赢不了，必须找独特切入点 |
| 衰退期 | 内容已饱和、讨论减少、新内容互动走低   | **一般放弃**。除非你有颠覆性新角度或总结性长文       |

### 判断方法

1. WebSearch `{话题}` → 看结果日期分布
2. 搜 `{话题} site:x.com` → 看最近讨论密度
3. 如果最新结果都是 3+ 天前 → 衰退期
4. 如果结果少且都是今天 → 萌芽期
5. 如果铺天盖地 → 爆发期

---

## Step 3: 角度空白分析

**关键动作**：搜已有内容，找出别人没写的角度。

### 执行方法

1. WebSearch `{话题}` → 读前 10 条结果的标题
2. 将已有角度归类：
   - 科普解释型（"什么是 X"）
   - 情绪反应型（"X 太离谱了"）
   - 实操建议型（"X 怎么应对"）
   - 深度分析型（"X 背后的逻辑"）
   - 个人经验型（"我亲历的 X"）
   - 反向观点型（"为什么 X 其实没那么糟"）
3. 找出**还没有人写的类型** = 你的机会

### 高价值空白角度信号

- 所有人都在骂 → 你写"为什么我理解他们"
- 所有人都在科普 → 你写亲身经历
- 所有人都在喊口号 → 你写具体数据分析
- 没人写实操 → 你写 step-by-step
- 都是宏观视角 → 你写一个小人物的故事

---

## Step 4: 定位匹配度评估

不是每个热点都适合你。评估 4 个维度：

| 维度     | 问题                                 | 低分信号                     |
| -------- | ------------------------------------ | ---------------------------- |
| 专业度   | 你对这个话题有真实认知或经验吗？     | 只看过新闻标题，没有一手体感 |
| 受众重合 | 你的目标读者关心这个话题吗？         | 话题虽火但和你的读者群无关   |
| 人设一致 | 写这个话题符合你一贯的风格和定位吗？ | 硬蹭、违和、画风突变         |
| 独特性   | 你能提供别人给不了的视角吗？         | 你的角度和头部博主一样       |

每个维度 1-3 分，总分 ≥ 8 分才值得写。

---

## Step 5: 综合评分 & 推荐

对每个候选话题打综合分：

```
话题评分 = 热度(权重25%) + 时机(权重30%) + 角度空白(权重25%) + 定位匹配(权重20%)
```

**时机权重最高**——因为好话题晚了就不值钱了。

## 特殊场景

### 用户自带选题

如果用户已有明确选题，跳过扫描，只执行：

- Step 2: 趋势阶段判断（确认时机）
- Step 3: 角度空白分析（找差异化）
- Step 4: 定位匹配度（确认值得写）

然后直接进入 BRIEF。

### 无明显热点

如果扫描后没有高分话题：

- 推荐"常青话题" — 不依赖热点但持续有流量的主题
- 建议从用户自身经历/专业出发，创造话题而非追话题
- 提示："今天不追热点也是一种策略"

### 多平台分发预判

在 DISCOVER 阶段就标注话题的平台适配性：

- 情绪型话题 → 微博/小红书优先
- 深度分析型 → 公众号/知乎/Newsletter
- 快速观点型 → X/Twitter thread
- 视觉化话题 → 小红书优先
